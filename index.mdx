`act` reads your `.github/workflows/` files and runs your GitHub Actions locally using Docker — giving you the same environment variables, filesystem structure, and execution model that GitHub provides on its hosted runners.

## How it works

<Steps>
  <Step title="act reads your workflows">
    act parses `.github/workflows/` and determines which jobs to run based on the event you specify.
  </Step>
  <Step title="Docker images are prepared">
    act pulls or builds the Docker images defined in your workflow files — matching GitHub's runner environments.
  </Step>
  <Step title="Jobs execute in containers">
    Each job runs in its own Docker container with the correct environment variables, secrets, and filesystem layout.
  </Step>
  <Step title="Results are streamed to your terminal">
    Step output is streamed in real time, with color-coded job names and log levels.
  </Step>
</Steps>

## Get started

```bash
# Install act (macOS)
brew install act

# Run the default push event
act

# List available workflows and jobs
act --list
```

<Note>
  act requires Docker to be installed and running. See the [installation guide](/installation) for platform-specific instructions and Docker setup.
</Note>
